## 2. 주간 리포트 생성 속도 개선 (`app.py`)

### 문제
기록이 많아질수록 주간 리포트 생성이 매우 느려짐 (기록 3개에도 ~190초 소요).

### 원인 분석
- 기존: 각 기록의 메모 전체 + AI요약 전체를 LLM에 전송
- 메모는 중복 정보 (AI요약이 이미 메모+이미지를 요약한 결과물)
- 불필요한 긴 입력 텍스트가 생성 시간 증가시킴

### 수정
메모를 완전히 제거하고, AI요약에서 `summary` 필드만 추출하여 전송.

```python
# 변경 전
entries_text += f"\n---\n날짜: {e['date']}\n제목: {e['title']}\n메모: {e['memo']}\nAI요약: {e.get('ai_summary','')}\n"

# 변경 후 — 메모 제거, AI요약 summary 필드만 사용
ai_raw = e.get('ai_summary', '')
ai_summary = ''
if ai_raw:
    try:
        ai_parsed = json.loads(ai_raw) if isinstance(ai_raw, str) else ai_raw
        ai_summary = ai_parsed.get('summary', '')
    except Exception:
        ai_summary = str(ai_raw)[:200]
entries_text += f"\n---\n날짜: {e['date']}\n제목: {e['title']}\nAI요약: {ai_summary}\n"
```

**파일:** `app.py` — `ai_weekly()` 함수
**주의:** 개별 AI 분석이 완료된 기록이어야 주간 리포트 품질이 보장됨.

## 3. 주간 리포트 항목 수 증가 (`app.py`)

### 문제
주간 리포트의 타임라인, 핵심 결과, 문제점, 다음 주 계획 항목이 너무 적게 생성됨 (각 1-2개 수준).

### 원인
프롬프트 예시가 1-2개뿐이라 LLM이 형식을 그대로 따라 최소한으로 생성.

### 수정
프롬프트에 최소 항목 수 명시 및 기록 개수 힌트 제공.

```python
"중요: 각 섹션에 항목을 충분히 채워줘. timeline은 위 기록 개수만큼 모두 포함, "
"key_results는 최소 4개 이상, problems는 최소 3개 이상 (없으면 기술적 도전/한계도 포함), "
"next_week는 최소 4개 이상 구체적으로 작성해줘.\n\n"
```

프롬프트 예시도 4개씩으로 확장:
```python
"  \"key_results\": [\"핵심 결과1\", \"핵심 결과2\", \"핵심 결과3\", \"핵심 결과4\"],\n"
"  \"problems\": [\"문제점1\", \"문제점2\", \"문제점3\"],\n"
"  \"next_week\": [\"다음 주 할 것1\", \"다음 주 할 것2\", \"다음 주 할 것3\", \"다음 주 할 것4\"],\n"
```

**파일:** `app.py` — `ai_weekly()` 함수 내 `prompt` 구성

## 4. 주간 리포트 레이아웃 수정 (`templates/index.html`)

### 문제
타임라인과 핵심 결과가 `grid-template-columns: 1fr 1fr`로 설정되어 있지만, 타임라인의 긴 텍스트가 컬럼을 밀어내어 50/50이 유지되지 않음.

### 원인
CSS grid에서 `1fr`은 콘텐츠 최소 너비를 보장하기 때문에, 내용이 긴 쪽이 더 넓어질 수 있음.

### 수정
```css
/* 변경 전 */
.weekly-grid { display: grid; grid-template-columns: 1fr 1fr; gap: 16px; margin-bottom: 20px; }

/* 변경 후 */
.weekly-grid { display: grid; grid-template-columns: minmax(0, 1fr) minmax(0, 1fr); gap: 16px; margin-bottom: 20px; }
.weekly-grid > .panel { min-width: 0; overflow-wrap: break-word; }
```

`minmax(0, 1fr)` + `min-width: 0`으로 콘텐츠가 길어도 정확히 50/50 유지.

**파일:** `templates/index.html` — CSS `.weekly-grid`

## 5. 재생성 버튼 텍스트 고착 버그 수정 (`templates/index.html`)

### 문제
주간 리포트 생성/재생성 완료 후에도 버튼이 "재생성중..." 스피너 상태로 고착됨.

### 원인
`generateWeeklyReport()` 성공/실패 핸들러에서 `regenBtn.innerHTML`을 원래 텍스트로 복원하는 코드 누락.

### 수정
```javascript
// 성공 시
if (regenBtn) { regenBtn.disabled = false; regenBtn.style.opacity = '1'; regenBtn.classList.remove('hidden'); regenBtn.innerHTML = '↺ 재생성'; }

// 실패 시
if (regenBtn) { regenBtn.disabled = false; regenBtn.style.opacity = '1'; regenBtn.classList.remove('hidden'); regenBtn.innerHTML = '↺ 재생성'; }
```

**파일:** `templates/index.html` — `generateWeeklyReport()` 함수
